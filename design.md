# Desktop Light Automation - 设计文档

## 1. 技术选型

### 选型: Python 3.9+

| 维度 | Python | Node.js |
|------|--------|---------|
| 子进程调用 | `subprocess` 标准库，简洁 | `child_process` 也不错 |
| 配置文件 | `json`/`yaml` 原生支持 | 需要安装依赖 |
| 日志系统 | `logging` 标准库，功能完善 | `winston` 等第三方库 |
| 定时任务 | `threading.Timer` / `time.sleep` | `setInterval` |
| 跨平台 | 更优（后续扩展考虑） | 稍弱 |
| 资源占用 | 轻量 | 稍重（V8 引擎） |

**结论**: 选择 Python，利用其完善的标准库实现零依赖（除 Python 本身）。

### 2.2 核心组件

| 组件 | 职责 | 文件 |
|------|------|------|
| ConfigManager | 加载、验证、提供配置 | `config.py` |
| StateManager | 读写状态文件，持久化 | `state.py` |
| Logger | 日志记录，按天轮转 | `logger.py` |
| Pinger | 执行 ping 命令，返回结果 | `pinger.py` |
| Debouncer | 防抖逻辑，状态判定 | `debouncer.py` |
| SceneTrigger | 调用 macOS 快捷指令 | `trigger.py` |
| Monitor | 主循环，协调各组件 | `monitor.py` |

## 5. 错误处理策略

### 5.1 错误分类

| 类别 | 示例 | 处理策略 |
|------|------|----------|
| 配置错误 | 配置文件缺失/格式错误 | **致命错误**，启动时退出，输出清晰错误信息 |
| 网络错误 | Ping 命令失败 | **可恢复错误**，记录警告，继续运行（视为离线） |
| 权限错误 | 无法写入状态文件 | **致命错误**，启动时退出 |
| 运行时错误 | 快捷指令调用失败 | **可恢复错误**，记录错误，30s 后重试 |
| 信号中断 | Ctrl+C | **优雅退出**，保存当前状态 |

### 5.2 异常处理原则

1. **配置加载阶段** - 严格校验，失败即退出
2. **主循环阶段** - 捕获所有异常，记录日志，继续运行
3. **状态持久化** - 原子写入（先写临时文件，再重命名）
4. **快捷指令调用** - 超时处理（10秒），失败不重试整个循环

### 5.3 日志级别

- **DEBUG**: 每次 ping 的原始结果
- **INFO**: 状态变化、场景触发
- **WARNING**: 单次 ping 失败、配置使用默认值
- **ERROR**: 快捷指令调用失败、文件写入失败
- **CRITICAL**: 配置错误、权限错误（退出前）

## 7. 启动脚本

```bash
#!/bin/bash
# scripts/run.sh

set -e

cd "$(dirname "$0")/.."

# 检查 Python
if ! command -v python3 &> /dev/null; then
    echo "Error: python3 not found"
    exit 1
fi

# 创建日志目录
mkdir -p logs

# 运行监控脚本
exec python3 src/monitor.py "$@"
```
