# Desktop Light Automation - 需求文档

## 项目概述

**目标**: 基于台式机在线状态自动触发 Apple Home 灯组开关场景

**核心场景**: 无窗房间，灯组专为台式机使用设计。台式机开启 = 人在使用 = 开灯；台式机关闭 = 人离开 = 关灯。

---

## 功能需求

### FR-1: 台式机状态检测

- **检测方式**: Ping 检测（ICMP）
- **检测目标**: 台式机固定 IP 地址（可配置）
- **检测频率**: 每 30 秒一次
- **防抖动机制**:
  - 连续 3 次在线 → 判定为"在线"
  - 连续 5 次离线 → 判定为"离线"

### FR-2: Apple Home 场景触发

- **触发方式**: macOS 快捷指令 CLI (`shortcuts run`)
- **场景**:
  - "到家"场景 → 开灯
  - "离开"场景 → 关灯
- **触发条件**:
  - 状态从"离线"变为"在线" → 触发"到家"
  - 状态从"在线"变为"离线" → 触发"离开"

### FR-3: 状态持久化

- **状态文件**: 记录当前台式机状态（在线/离线）
- **位置**: 项目目录内的 `state.json`
- **目的**: 脚本重启后不重复触发

### FR-4: 配置管理

- **配置文件**: `config.json`
- **可配置项**:
  - 台式机 IP 地址
  - Ping 间隔（秒）
  - 在线判定阈值（连续次数）
  - 离线判定阈值（连续次数）
  - 快捷指令名称（到家/离开）

### FR-5: 日志记录

- **日志文件**: `logs/monitor.log`
- **日志内容**:
  - 每次 Ping 结果
  - 状态变化事件
  - 场景触发事件
  - 错误信息

---

## 非功能需求

### NFR-1: 可靠性

- 脚本需具备自恢复能力（崩溃后可重启继续运行）
- 网络抖动不导致误触发

### NFR-2: 可维护性

- 配置与代码分离
- 日志完善，便于排查问题

### NFR-3: 轻量级

- 资源占用低（持续运行）
- 无外部依赖（使用 macOS 原生工具）

---

## 技术约束

- **运行环境**: macOS（OpenClaw 所在的 Mac）
- **编程语言**: 优先 Python 或 Node.js（阿尔萨斯决定）
- **依赖**:
  - macOS 快捷指令 CLI（`shortcuts` 命令）
  - 标准网络工具（`ping`）

## 验收标准

1. ✅ 脚本持续运行，每 30 秒检测一次
2. ✅ 台式机开机后 3 分钟内自动开灯
3. ✅ 台式机关机后 5 分钟内自动关灯
4. ✅ 网络抖动（单次 Ping 失败）不触发误动作
5. ✅ 脚本重启后不重复触发场景
6. ✅ 日志清晰可查
